# Blog Conversion — 옵시디언 노트 → 블로그 변환

옵시디언 노트를 네이버 블로그·티스토리 같은 *독자용 글*로 재작성하는 가이드.
**원칙**: 같은 데이터, 다른 톤. 노트의 *결정체*만 추출.

---

## 옵시디언 vs 블로그 — 차이의 핵심

| 옵시디언 (1차) | 블로그 (2차) |
|---------|-------|
| 무거움, 자기대화 | 가벼움, 독자 호출 |
| 길고 진중 | 짧고 펀치 |
| "지금 생각해보니" 같은 자기 검토 | "근데 진짜~", "이게 뭐냐면" |
| 4차원 평점 본문 인라인 | 평점 X (또는 추천/비추천) |
| 분석 + 감정 다 풀음 | *가장 박힌* 1~3개만 |
| 한 글에 모든 차원 | 글 분량 = 노트의 1/3~2/5 |
| h1·h2 구조 위계 | h2 큰 섹션 + 자유로운 강조 |
| 인용 콜아웃 | 이미지 친화 (`[Image: ...]` 자리) |
| 제목: 작품명 그대로 | 후킹 제목 3안 제안 |

---

## 개인 블로그 톤 — 핵심 규칙

### ✅ 살릴 것

**호흡**
- 한 단락 1~2문장으로 압축
- 빈 줄 자주 넣어서 시각적 호흡 짧게
- 한 호흡으로 길게 쓴 분석체 ❌ → 짧게 끊어서 ✅

**정중체 + 반말 자유로운 섞기**
- "~합니다", "~이에요", "~인데요"
- "~거든요", "~더라고요"
- "이게 뭐냐면", "근데 진짜", "이게 맞나?"
- "암튼", "별 건 없는데 굳이 꼽자면"

**강도 있는 감탄 (개인 시그니처)**
- "미쳤다", "지렸다", "독보적임"
- "강추"
- "이만큼 ~한 게임/책/영화가 있을까"
- 단, **감탄 직후 *왜* 그런지 1단락 따라옴**. 감탄만 던지고 끝내면 AI식.

**약한 욕설 (강조할 때만)**
- "개", "ㅈㄴ" — OK, 1~2번
- ❌ 절대 금지: "시발", "씨발", "병신", 그 외 강한 욕

**옆길 빠지기**
- 한 글에 0~1번. 너무 엇나가지 않게.
- 짧게, 본론으로 빠르게 복귀.

### ❌ 뺄 것

**em dash (`—`)**
- 한국 글에선 거의 안 씀. AI 티 가장 강함.
- 대체: 마침표로 끊거나, 괄호 ( ), 쉼표

**평면적 형용사 (알맹이 없음)**
- "진짜 좋은 게임이었습니다"
- "정말 재밌었어요"
- "100시간 박았는데 아깝지 않음"
- "추천드립니다"
- → 이건 어떤 작품에든 갖다 붙일 수 있는 클리셰. *근거 없는 감탄*은 AI식.

**이모지·과한 이모티콘**
- ❌ 진짜 이모지 (🎮 🔥 ✨ 등)
- ❌ `^^`, `;;`, `ㅠ`, `ㅎ` 자주
- ✅ `^^` 1번까지 (사용자의 "용서했다 ^^" 같은 자조 자리)
- ✅ `ㅋㅋ` 1~2번까지

**따라하기 시그니처 (다른 블로거 흉내 X)**
- 후루꾸의 `레지고`, `끝.` ← 이거 쓰면 따라하는 티
- 가상의 캐릭터 대화 ("이 문과색히야!!!") ← 개인 스타일 X

## 변환 워크플로우

### Step 1 — 후킹 제목 3안 제안

옵시디언 노트 제목은 보통 작품명. 블로그는 *후킹*이 필요.

3안 제시:
- **A. 임팩트 직격**: "100시간 했더니 체코 가고 싶어졌다 - KCD2 후기"
- **B. 후루꾸식 평이**: "스포 없음) 진짜로 중세에서 살아봤음 - KCD2"
- **C. 캐릭터/메시지 호출**: "헨리야 너 진짜 멋있는 기사가 됐구나"

→ 사용자가 고르거나 새로 제안.

### Step 2 — 결정체 추출

노트에서 *블로그에 살릴 것*만 표시:

| 노트 섹션 | 블로그 추출 기준 |
|---------|---------------|
| 첫인상 | 도입 한 줄 |
| 작품의 골격 | 압축해서 "어떤 작품이냐" |
| 좋았던 점 | 가장 두꺼운 2~4개만 |
| 아쉬웠던 점 | 그대로 (단, 짧게) |
| 분석 / craft | 좋았던 점에 통합 |
| 인상적인 순간 | 1개만 (가장 박힌 것) |
| 나에게 남긴 것 | **이건 무조건 살림. 글의 임팩트 자리** |
| 4차원 평점 | 본문에서 빼고, 추천/비추천으로 변환 |
| 평점 근거 | 좋았던 점/아쉬운 점에 녹임 |
| 함께 읽기 / 함께 플레이 | 추신으로 |

### Step 3 — 개인 톤으로 재작성

위 "개인 블로그 톤" 규칙 따름. 특히:
- em dash 0개
- 평면 형용사 빼기
- 정중체 + 반말 섞기
- 호흡 짧게

### Step 4 — 이미지 자리 표시

```
[Image: 어떤 사진이 들어가면 좋을지 설명]
*캡션 한 줄*
```

배치:
- 글 맨 위: 메인 비주얼 (포스터, 메인 컷)
- 큰 섹션마다 1개: 관련 스샷
- 캡션은 짧게, 한 줄

### Step 5 — 추신 마무리

`+`, `++`, `+++` 형식. 본문에 안 들어간 짧은 코멘트.
"끝." 같은 단정적 마무리 ❌ (따라하기). 추신으로 자연스럽게 페이드 아웃.

## 플랫폼별 미세 조정

### 네이버 블로그
- 줄바꿈이 *시각적으로* 매우 중요. 빈 줄 더 자주.
- 이미지 자리 명확히. 사진 캡션 한 줄.
- `**굵게**`는 잘 작동, 코드 블록은 안 어울림.

### 티스토리
- 마크다운 그대로 작동. 코드 블록 OK.
- 줄바꿈은 네이버보다 덜 빠듯해도 됨.

### 일반 (벨로그·미디엄·범용 마크다운)
- 그대로.

---

## 안티패턴 (반드시 피할 것)

❌ 노트 통째로 복붙 후 톤만 바꾸기 → 너무 길어짐
❌ 노트의 4차원 평점을 그대로 옮기기 → 블로그 톤과 안 맞음
❌ "오늘은 ~에 대해 알아보겠습니다" → AI 티
❌ "여러분 안녕하세요" 클리셰 도입 → 후루꾸식이 아니라 *사용자의 자연스러운* 도입
❌ em dash 한 번이라도 사용 → AI 티 가장 강함
❌ 모든 섹션 끝에 "정말 좋은 ~입니다" 결론 → 평면 형용사

## 변환 예시 (KCD2 케이스)

| 노트 원본 | 블로그 변환 |
|---------|-----------|
| "복수라는 감정이 결국 파멸과 공허로 남긴 하지만, 그로 인한 헨리의 과정을 세세히 만들어 갔던 거 같다." | "게임이 보여주는 핵심 메시지는, **복수가 결국 파멸과 공허로 남긴다**는 거예요. 근데 이게 그냥 '복수는 나쁩니다' 같은 일차원 메시지가 아니라, 헨리가 *그 과정을 직접 겪게* 만들면서 보여주는 식입니다." |
| 4차원 평점 4줄 + 근거 | "**무조건 추천**: 시대물 좋아하는 사람... / **비추천**: 화려한 액션 원하는 사람..." |
| `## 함께 플레이하기` | `\+ 1편 했으면 무조건 / \++ 빌드 다양성 부족` |
