# 책 노트 템플릿

## 4차원 해석 (책)

| 차원 | 책에서의 의미 |
|------|--------------|
| **craft** | 문체·언어·문장 구성·번역 (번역서일 경우) |
| **narrative** | 논지 전개·서사 구조·메시지 |
| **impact** | 사유를 자극한 정도·새로운 관점 제공 |
| **rewatch** | 재독 가치 / 다시 펼치고 싶은가 |

## Phase 1 질문 셋 (책)

### 공통 — 항상 포함

```
1. 줄거리 / 책의 골격 한 단락
2. 다 읽고 가장 먼저 떠오른 감정 한 단어 + 왜
3. 가장 인상적인 순간 (구절·관점·전환점)
4. 좋았던 점 — 떠오르는 대로
5. 아쉬웠던 점 — 있으면
```

### 매체 특화 — 책

```
B1. 문체에 대한 인상 (건조? 시적? 학술적? 도발적? 등)
B2. 마음에 박힌 구절 — 그대로 옮겨와줘 (페이지·위치도)
B3. 저자의 사고 방식이 어땠어? (논리적? 직관적? 도발적? 친절한?)
B4. 책 도중 멈춰서 생각한 적 있어? 어디서?
B5. 다 읽은 후, 책에 *동의*하는 부분과 *동의 안 하는* 부분
B6. (해당 시) 이 책이 너의 기존 생각을 바꿨어, 강화했어, 혼란시켰어?
```

## 본문 구조

```markdown
# {{title}}

## 한 줄 요약

> 사용자가 직접 정리한 한 문장 (Phase 1-1)

---

## 첫인상

> Phase 1-2 + Phase 0 답변 기반.
> 다 읽고 가장 먼저 떠오른 감정 + 한 단어로 표현된 결.

---

## 줄거리·논지

> 책의 핵심 주장 또는 서사를 사용자의 언어로 정리.
> Claude가 채우지 말 것 — 사용자가 한 정리만 사용.

---

## 좋았던 점

> 단순 나열 ❌. 각 항목마다 *왜* 좋았는지 한 줄.

---

## 아쉬웠던 점

> 있으면. 없으면 섹션 생략.

---

## 인상적인 구절

> Phase B2에서 옮긴 구절들.
> 각 구절 아래에 *왜 박혔는지* 사용자의 코멘트.

```
> "구절 원문"
> 
> — 페이지 / 챕터
> 
> 사용자의 반응: ...
```

---

## 분석

> 책의 논리 구조·문체·논증 방식에 대한 사용자의 분석.
> Phase 2에서 꺼낸 "왜?"의 결과물.

---

## 나에게 남긴 것

> impact 차원의 본문.
> 감정 어휘 + 사용자의 사유 변화·강화·동요.
> 책이 사용자의 어떤 부분을 건드렸는가.

---

## 동의 / 비동의

> Phase B5 답변 기반.
> 책의 어디에 동의하고, 어디에 동의하지 않는지.
> 비동의 지점이 곧 사용자의 사유 위치를 드러냄.
```

## 작성 노트

- 인용은 `>` 콜아웃으로
- 페이지 번호 있으면 명시
- 저자명·번역자명은 옵시디언 위키링크로 연결: `[[유발 하라리]]`
- 동일 저자의 다른 책 노트가 있으면 본문에 위키링크: `[[호모 데우스]]`
- 비슷한 결의 다른 책이 떠오르면 마지막에 `## 함께 읽기`로 연결
